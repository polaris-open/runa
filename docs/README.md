# Runa documentation

Documentation for **Runa — Draft v0.1 (runnable skeleton)**. These docs describe
both what exists today and where the project intends to go. Where something does
not exist yet, the docs say so.

## Index

- [vision.md](vision.md) — why Runa exists.
- [principles.md](principles.md) — the design principles, explained.
- [terminology.md](terminology.md) — shared vocabulary.
- [architecture.md](architecture.md) — layers and flow.
- [mvp.md](mvp.md) — what v0.1 is and isn't; how to validate it.
- [configuration.md](configuration.md) — the `runa.yaml` file.
- [vault-contract.md](vault-contract.md) — what Runa expects of a vault.
- [proposals.md](proposals.md) — the proposal-first model.
- [providers.md](providers.md) — the provider-neutral plan.
- [personas.md](personas.md) — personas as configuration.
- [safety-and-privacy.md](safety-and-privacy.md) — safety and privacy stance.
- [decisions/](decisions/README.md) — where design decisions will be recorded.

## How to read these

If you only read three documents, read [vision.md](vision.md),
[principles.md](principles.md), and [mvp.md](mvp.md). They explain *why* Runa
exists, *how* it should behave, and *what* is actually built right now.
